### Analytics conclusion

- A phase-1 first-party GA4 stack now exists in repo code.
- Product decision for now: leave GTM out of this project.
- For Positives, GTM would only be useful later if the team wants non-engineering control over marketing and ad-platform tags.
- If marketing attribution matters, the better starting point is a first-party event taxonomy in code; GTM can be layered on later for the marketing site only.
- The current phase-1 implementation is intentionally focused and meaningful, not broad:
  - acquisition: `page_view`, `join_cta_clicked`, `referral_visit_detected`
  - conversion: `begin_checkout`, `purchase`
  - activation: `sign_up`, `account_activated`, `affiliate_enrollment_completed`
  - retention and revenue health: `billing_portal_opened`, `upgrade_completed`, `subscription_canceled`, `payment_failed`
  - support and friction: `support_submitted`, `support_submit_failed`, `checkout_error`
- Server-side purchase and subscription events are sent through GA4 Measurement Protocol from Stripe webhook handlers, so there is no separate Stripe-to-GA dashboard integration required.
- GA4 is configured with:
  - property: `Positives`
  - web stream: `https://positives.life`
  - measurement ID: `G-0VMX9MPSDY`
  - reporting timezone: `America/New_York`
- Production rollout is still incomplete:
  - local env has the GA keys
  - Vercel production does not yet have `NEXT_PUBLIC_GA_MEASUREMENT_ID`
  - Vercel production does not yet have `GA_MEASUREMENT_PROTOCOL_API_SECRET`
- The goal should be meaningful information, not event spam. Track a short list of events tied to actual decisions:
  - acquisition: landing CTA click, join CTA click, referral-attributed visit
  - conversion: checkout started, checkout completed, refunded, canceled
  - activation: first successful login, first practice completed, first affiliate enrollment
  - retention: return after 7 days, billing recovery, upgrade started, upgrade completed
  - support and friction: support form submitted, billing portal opened, affiliate-link creation failed
- Every tracked event should have a clear owner, a business question it answers, and a destination for review. If an event does not change a decision, it should probably not exist.
- Current privacy copy says the site does not use advertising or tracking cookies, so GA4 rollout now requires privacy-policy review before or alongside production deployment.
