# Beta Issue Severity Rubric

## Goal

Make beta triage consistent so everyone uses the same language when deciding what needs immediate action.

## Blocker

Use when:

- a tester cannot sign up, sign in, pay, or access core content
- a page crashes consistently
- billing or access is wrong in a way that risks trust or revenue
- data loss or a serious permissions problem is involved

Expected response:

- same-day review
- immediate owner assignment
- track through to verified fix or workaround

## High

Use when:

- a core flow still works, but with major friction
- a key launch promise is undermined
- a bug affects multiple testers or an important segment
- the issue creates strong confusion, support drag, or retention risk

Expected response:

- prioritize into current active work
- communicate workaround if needed

## Medium

Use when:

- the issue is real and worth fixing, but not blocking usage
- UX confusion slows people down without fully breaking the flow
- content or workflow clarity needs improvement

Expected response:

- triage and assign
- fix in normal beta iteration

## Low

Use when:

- polish issue
- isolated edge case
- copy/style/layout refinement
- idea or enhancement that is useful but not urgent

Expected response:

- capture and batch
- roll into later polish or post-beta roadmap if needed
