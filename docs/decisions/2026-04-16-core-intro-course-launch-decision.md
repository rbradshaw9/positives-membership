# Core Introductory Course At Launch

## Summary

Recommendation:

- **Do not make a core introductory course part of the launch promise.**
- If we want a gentler on-ramp later, add it as an **optional “Start Here” layer**, not as the center of the product.

This keeps Positives aligned with its actual strength:

- a daily practice
- a returning rhythm
- a calmer member experience

It also avoids teaching members to think of Positives as a course portal they are supposed to complete.

## Why A Core Course Is Not The Right Launch Move

### 1. It weakens the core product story

Positives is strongest when it feels like:

- a daily mindset practice
- a steady emotional reset
- something you return to

Not:

- a curriculum
- a lesson sequence
- a course you finish before the “real” value begins

If launch messaging suddenly says every member should start with a core course, we undermine the clearest product truth we already have.

### 2. It adds content pressure without solving the main launch risk

The main launch challenge is not “members need more content.”

It is:

- members need to understand the daily rhythm quickly
- members need to come back after the first day
- members need the live/event/coaching promises to be truthful

A new core course creates content work, structure work, copy work, and onboarding complexity without obviously improving those launch risks enough.

### 3. It can create a “behind” feeling

One of the strongest Positives principles is that members should not feel behind.

A launch-course layer risks creating exactly that:

- “I should finish the starter course first”
- “I haven’t completed onboarding yet”
- “I’m already behind”

That is the opposite of the emotional posture the product should teach.

## Better Launch Alternative

Instead of a core course, launch with:

- a strong welcome sequence
- a clear Today page
- a clear Practice page
- a simple explanation of the daily / weekly / monthly rhythm
- optional course purchases or bonus learning as an add-on, not the center

The best onboarding at launch is probably:

1. Start with today’s practice.
2. Understand how the weekly and monthly rhythm works.
3. Come back tomorrow.

That is enough.

## If We Want A Learning Layer Later

A better later model would be:

- an optional `Start Here` collection
- a short guided path like `Your first 5 days`
- a small orientation series inside Library or Practice
- possibly a lightweight onboarding journey, not a formal course

Important constraint:

- it should support the daily habit
- it should not compete with the daily habit

## What To Keep At Launch

At launch, the product should center:

- `Today`
- `This Week`
- `This Month`
- events for Level 2
- coaching for Level 3
- optional purchased courses as separate products

Courses can absolutely exist at launch.

But they should feel like:

- optional deeper dives
- separate purchases or special resources
- not the mandatory front door to Positives

## Recommended Decision

Approve this launch posture:

- no core introductory course as part of the main membership launch
- onboarding is handled through product structure, welcome messaging, and daily rhythm
- optional course-based orientation can be explored later only if it clearly improves activation without turning Positives into a course product

## Follow-On If Approved

If approved, we should:

- keep homepage and join messaging centered on the daily practice
- avoid “complete the program” framing in onboarding copy
- treat course commerce as a separate growth/revenue layer
- revisit an optional `Start Here` path only after launch data shows a real need
